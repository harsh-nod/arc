# AIR Type System

## Goals

- Provide dependent indices to capture shapes, bounds, and policies.
- Distinguish between value, resource, proof, authority, and policy types.
- Remain decidable for automated verification through a restricted index language.

## Base Types

- `iN`, `uN`, `f32`, `f64`, `bool`
- `!air.mem`, `!air.world`, `!air.fs`, `!air.net`, `!air.db`, `!air.clock`, `!air.rng`, `!air.ui`, `!air.gpu`, `!air.vault`
- `index` values used for dependent expressions.

## Composite Types

- Tuples: `(!air.mem, i64)`
- Function types: `(!air.mem, !air.ptr<i64>) -> !air.mem`
- Struct-like records defined by dialects.
- Resources annotate ownership and linear usage requirements.

## Indexed Types

Syntax: `!air.tensor<f32, [%m, %n]>`

Indices come from the index language:

```
expr ::= integer
       | symbol
       | expr + expr
       | expr - expr
       | expr * constant
       | expr / constant
       | min(expr, expr)
       | max(expr, expr)
```

Equality uses canonical normalization plus SMT fallback for linear arithmetic.

## Refinement Types

Refinements express predicates scoped by proofs.

Examples:

- `!air.proof<%i < %n>`
- `!air.slice<i32, %n>`
- `!air.tensor<f32, [%m, %k]>`
- `!air.auth<action = @email.send, resource = %draft, valid_until = %deadline>`

Refinements must be justified by proof operations or runtime checks generated by the verifier.

## Proof Types

Proof types encode logical obligations verified statically or discharged dynamically.
Proofs are first-class values that may be erased after dominance analysis.

Key operations:

- `air.assume` to introduce axioms (restricted to trusted contexts).
- `air.assert` to insert runtime proof obligations.
- `air.prove` to discharge goals via solver or tactic.
- `air.refine` to strengthen types given proofs.
- `air.pack`/`air.unpack` for existential packages.

## Authority Types

Authority tokens are linear values:

```
!air.auth<principal = %user,
          action = @email.send,
          resource = %draft,
          valid_until = %deadline>
```

They are consumed when invoking capabilities and must dominate the use site.

## Existential Packages

```
exists (%n: index). !air.slice<i32, %n>
```

Packages are unpacked with `air.unpack` which binds the witness symbol.

## Type Equivalence

- Alpha-equivalence on bound indices.
- Normalization of simple arithmetic expressions.
- Refinement entailment delegated to solver or proof kernel.

## Type Erasure

- Value types erase to runtime representations (primitive scalars, aggregates).
- Resource, proof, and authority types have dedicated runtime representations.
- Erasure respects effect and policy contracts; proofs may be replaced with runtime checks.
